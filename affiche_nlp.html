1 — Le problème

Avez-vous déjà essayé d'apprendre le japonais, sans savoir si un texte est adapté à votre niveau ?

Le JLPT (Japanese Language Proficiency Test) définit 5 niveaux de maîtrise :

N5 débutant N4 N3 N2 N1 avancé

Un texte japonais combine kanji, hiragana, katakana et grammaire à des niveaux variés, rendant l'évaluation manuelle difficile.

Objectif : prédire automatiquement la difficulté en 3 classes :

easy medium hard

3 — Régression Logistique

Le classifieur utilisé est une Régression Logistique multiclasse . En 3 classes (easy, medium, hard), on calcule d’abord un score par classe k

La validation finale s'effectue sur le jeu de validation séparé.

zk = wk⊤x + bk

Avec k ∈ {easy, medium, hard}, x = vecteur de features, (wk, bk) = paramètres appris pour la classe k.

Puis on obtient une distribution de probabilités par la softmax :

P(y=k | x) = ezk / ∑j ezj
Chaque P(y=k|x) est dans [0, 1] et représente la confiance du modèle pour la classe k.

La classe affichée (Prediction: ...) est celle de probabilité maximale (argmax) :

ŷ = argmaxk P(y=k | x)

En multi-classes, softmax normalise pour que la somme fasse 1:

P(easy) + P(medium) + P(hard) = 1

La classe prédite est celle de probabilité maximale (argmax). Un score 0–100 est ensuite calculé à partir de P(easy) pour résumer la facilité du texte.

Score = round(P(easy) × 100)
4 — Extraction de features linguistiques

Chaque texte est converti en un vecteur numérique explicable :

Ratios d'écriture
ratio_kanji ratio_hira ratio_kata
Structure
punct_count sent_count comma_count
Signaux JLPT vocab
vocab_ratio_N5 vocab_ratio_N3 vocab_ratio_N1
Kanji + Grammaire
kanji_max_level grammar_ratio_N2 kanji_ratio_N1

La normalisation NFKC est appliquée avant extraction. C'est une forme de normalisation Unicode qui réduit les variantes spécifique : ｶﾀｶﾅ ↔ カタカナ

Score 0–100 : synthèse de la probabilité softmax de la classe easy.

Score = round(P(easy) × 100)
Distribution du score 0–100 par label réel — séparation nette entre les classes

7 — Limites et améliorations

Principale limite : la classe medium est la plus difficile à capturer :

Elle se situe naturellement entre easy et hard
Les features "proportions JLPT" peuvent manquer de signal sur la forme (style, tournures)
Un corpus petit ou bruité amplifie l'instabilité de cette classe

Techniques d'amélioration du modèle :

Lemmatisation (SudachiPy) : les listes JLPT sont en forme dictionnaire (ex. 食べる), alors que le texte contient des formes conjuguées (食べた, 食べます…). SudachiPy extrait la forme de base de chaque token pour mieux matcher le vocabulaire JLPT et améliorer la qualité des features. Sans SudachiPy, des heuristiques de déconjugaison prennent le relais (〜ます → forme en る, etc.).
Stress-test : génère des phrases artificielles en répétant un terme JLPT ciblé (ex. 無料) pour vérifier que le modèle ne sur-prédit pas la difficulté d'un seul mot. Les résultats sont exportés dans un CSV pour analyse.
8 — Continuité

Par la suite, il sera possible de :

Affiner la granularité : passer de 3 classes à 5 (N5→N1) pour coller aux niveaux JLPT officiels
Déployer une interface web permettant de coller un texte et d'obtenir son niveau estimé en temps réel
Intégrer des modèles de langue pré-entraînés (BERT japonais — cl-tohoku/bert-base-japanese) pour capturer le contexte sémantique
Enrichir le corpus avec des textes annotés publics (NHK, NHK Web Easy,etc..)
